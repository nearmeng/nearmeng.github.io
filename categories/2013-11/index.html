git使用介绍

1、安装

首先下载git软件安装，点此下载。

在安装完git之后，打开Git Bash的快捷方式，进入命令行模式。

2、配置

首先我们需要创建一个git仓库的文件夹。

mkdir git

接着创建一个你的工程文件夹，例如是project.

cd git && mkdir project

然后我们进入git文件夹，进行初始化操作。

cd git
git init
git config --global user.name "yourname"
git config --global user.email "youremail"

此时将会在你的git文件夹下生成.git文件，该文件的作用是将会对你上传的代码文件进行取舍，将一些调试用的文件排除。

3、下载

此后，可以进入你的project文件夹，根据远程git仓库的地址，clone一组代码回来。

git clone https://git.oschina.net/.../...git

或者你可以把你的文件放在project文件夹下，然后，和远端git仓库中的代码进行合并。

git pull https://git.oschina.net/.../...git

4、上传

当你修改完你project中的文件后，你需要首先将你的改动提交到你的本地的仓库中。

git add .
git commit -m "your comments"

然后就可以push到你的远程的仓库中（在这之前，首先需要增加一个远程仓库的位置）。

git remote add origin https://git.oschina.net/.../...git
git push -u origin master

此后，会让你填写你的git@osc的用户名和密码。

5、注意及其他

5.1 需要更改你的分支名

一般你的文件夹下你的分支名是master，而如果你要更改分支名的话。使用以下命令。

git branch /* 查看当前的分支 */
git branch hello /* 创建一个名为hello的分支 */
git checkout hello /* 将当前的分支改为hello分支 */

5.2 git 报错

（错误1） Updates were rejected because the tip of your current branch is behind its remote counterpart. Merge the remote changes （e.g. “”git pull“”） before pushing again.

解决方法： 首先在当前的project下，进行一次

git pull  https://git.oschina.net/.../...git

然后再进行commit以及push操作。

（错误2） warning: LF will be replaced by CRLF

解决方法： 这是由于Windows和Linux的文件结束符不一致导致的。需要删除.git，并重新配置。

rm -rf .git  
git config --gobal core.autocrlf false

接着就可以重新 init 了。

markdown语法简介

已经好久没有开始写博客了，自从上次的博客域名expire之后，就没有了写博客的冲动。最近，突然又想拾起笔来，重新做人:) 于是就开始找寻可以记录信息的地方，大大小小的博客网站虽然多，但是能够还自己一份安宁的地方却是少之又少，就在偶然的查找资料的过程中，我发现了 scriptogr.am 这个网站，可以说真是缘分，它使用的是我一直使用的 Dropbox 服务，只需要在Dropbox中存放你自己编写的md格式的文档，经过scriptogr同步之后，就可以将md格式转换为html输出，变成你博客的一篇文章。

网站的界面清爽简洁，也真是我一直希望的博客的格式，节约却不简单，哈哈。在熟悉了博客的环境之后，我去熟悉了以下传说中的简单的MarkDown文本编辑语言，这是一种非常易读易写的文本编辑语言，有关于它的介绍可以这里查看->,它的学习和使用也是非常的简单和人性化，使用教程可以在这里查看->,借用网上的一句话说，那就是，这是一种你可以完全掌握的语言。学习Markdown语言，主要需要记住的几个语法分别是：

文章换行

敲两个回车(或者是先敲两个以上的空格再加个回车)

文章标题

#   一级标题
##  二级标题
### 三级标题 等

文本标注

*A* A倾斜
**A** A加重

文字添加链接

[nearmeng](scriptogr.am/nearmeng "optional") 添加一个nearmeng的链接
<http://scriptogr.am/nearmeng>   生成一个URL超链接
<nearmeng@gmail.com>             生成一个邮箱链接

添加图片

![pic](http://pic_path "pic name")

引用一段文字

使用'>'即可 符合邮件的规范

引用一段代码

代码块可以使用一个Tab或者四个空格，即可输入代码
单纯的一行代码可以使用反引号实现，即"`"

水平横线

直接使用"***"即可生成

有序无序列表

无序列表： 在一个空行之后，使用*分别列表显示
有序列表： 在一个空行之后，使用1.分别列表显示

在经过短暂的学习之后，就可以很熟练地使用MarkDown语言进行文本的编辑了，基本不需要工具，在这里，我使用的是Windows下的MarkdownPad进行编写，Linux下使用的是Vim进行编写文档。

自己写博客，主要也就是基于如下的考量：

  • 记录生活中的点滴
  • 提高自己梳理和总结的能力
  • 提高自己语言表达的能力
  • 对所学习的知识进行记录，思考以至深化

同时，现在的网络通信讲究的是共享精神，网络也是在共享知识的基础上发展起来的，所以知识的交流对于个人的提高就显得更为重要，闭门造车会显得既老土又单调。

所以，加油吧，少年，希望自己可以养成勤于记录和博客的习惯，在微博，人人，qq盛行的浮夸和娱乐的年代，能够开辟一方净土，安静地耕耘。

是以为记。
